What just happened

Open console in debugging tool of your browser (usually by pressing Ctrl +Shft+I ). You should see the following messages: 

            Hello from MyES6Class constructor!          myES6Class.js:3
            Hello ignorant!                             myES6Class.js:7
            Hello from the static function!             myES6Class.js:11
            [WDS] Live Reloading enabled.               client:52
  • The first message comes from the constructor of the ES6 class that is created in the path ./src/myES6Class.js
  • The second one comes from its greet() method.
  • The third message is due to its static function myStaticFunction()
  • The fourth message can be ignored (since we are ignorant 😎). It is coming from Webpack Development Server.
All of these got wired-up in ./src/index.js which creates an instance of the previously mentioned class and plays around.

Live reloading

Already enabled (as the fourth messaged indicated). Tweek some codes in the files of ./src folder and you should see the changes (almost) instanteniously.

Styling

Notice that the styling for this page lives in ./src/style.css file which is imported in the ./src/index.js file using an ES6 import syntax. This ignorable magic happens because of a webpack loader. Edit ./webpack.config.js if you know what you are doing.

Debugging

Open Debugging tool of your browser, press Ctrl+p, type the name of your source file and press Enter. An exact replica of your source file should be visible in the Sources tab of the debugger. Set break points and you know how to debug the hell out of it. Agian, this ignorable magic happens because of the line, devtool: 'source-map' in webpack.config.js
